Static Site Generation with Hugo

May 30, 2020

2 minute read

 LinkedIn

 Email

So I had issues with getting Pelican to work on Windows (I game a lot, so Windows is more practical for me than a nix distribution), mostly with theme generation because I really didn’t want to have to learn yet another language, just to get a website up and running. I recently learned HTML5 and relatively modern CSS practices, including responsive design, and I didn’t particularly feel like branching out again so soon.

So what I decided to do was find a theme that I liked, and go with the static generator associated with that theme. I ended up landing on Hugo Future Imperfect, a theme which seems to have a rich history as it has many project contributors. Hugo is a static site generator based on Goland, and is compatible with the Windows command terminal. As I am interested in learning Go in the future, this seemed like a no brainer to me. As is often the case when I take things on, using this site-generator in no way lets me begin to learn Go, so that remains in The List.

Installation was easy, and some minor modifications to the theme (and some git shenanigans) later, I have a sleek, modern static site for my blog! Wonderful!

Now to actually generate content for it.
